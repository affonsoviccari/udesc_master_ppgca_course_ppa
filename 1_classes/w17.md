# PCAM - Mandel
## P
Evitar poucas tarefas.
1. Z por task (!= thread).
2. Filtro, pré processamento para balanceamento
3. Bloco
4. Linha/Coluna

## C
Evitar a movimentação de memória (p*)
2. Majoritatiamente local e estruturada, estática. Pois para cada task, se utilziada o (P: 1), basta passar qual a linha e coluna, não precisa passar toda a matriz.

## A
Evitar a troca de contexto.
1. Agrupamento por linhas, para reduzir a troca de contextos, uma vez que número de tarefas precisa ser >= ao número de processadores.
> Podemos ter infitinas tasks, porém haverá muita troca de contexto, aumentando o tempo de processamento (ou não reduzindo).

## M
Evitar o desbalanceamento.
1. Banco de tarefas
O número de tarefas permanece maior que o número de processos (workes).
Workpool, fazer o mapeamento das tarefas nos processos sem a sobrecarga de movimentação de memória e troca de contexto.
Mapeamento de N:P (tarefas:processador) de forma dinâmica, pois o tempo de processamento das tasks é diferente.


## Código - Mandel
Pode se escrever:
```C
	char **mat = (char**)malloc(sizeof(char*)*max_row);
	for (int i=0; i<max_row;i++)
		mat[i]=(char*)malloc(sizeof(char)*max_column);
```
ou em bloco:
```C
    char *bloco = (char*)malloc(sizeof(char)*max_row*max_column)
	for (int i=0; i<max_row;i++)
		mat[i]=&bloco[i*max_columns];
```
Impacto em processamento local, pois a memória é alinhada uma única vez na memória.


```C
#pragma omp parallel for num_threads(10) schedule(dynamic)
```
Não é possível definir `num_threads(10)` no #pragma omp parallel for, então é preciso fazer um workaround:
```C
#pragma omp parallel num_threads(10)
{
    #pragma omp for schedule(dynamic)
    for (...)
        for (...)

    #pragma omp single
    {
        // do single time
    }
}
```

Pode-se utilizar `export OMP_NUM_THREADS=1` (linux) ou `set OMP_NUM_THREADS=1` (windows)
